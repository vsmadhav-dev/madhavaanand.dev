Profile

I'm a student developer with a particular interest in how things work underneath — memory, file I/O, data structures — and how that same discipline carries over into building interfaces that are fast and easy to use. Most of my projects split cleanly into two buckets: C programs that manage and persist data reliably, and front-end builds that focus on semantic markup and responsive layout rather than frameworks.

I like keeping code simple enough to read a year later, and I'm always looking for projects that push me to learn the next layer down or up the stack.
